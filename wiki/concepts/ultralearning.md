# Ultralearning

## Definition

An intense, self-directed approach to skill acquisition that emphasizes planning how to learn, practicing the target skill directly, isolating weaknesses, testing recall, seeking feedback, and iterating beyond basic competence.

## Detail

In the current source set, ultralearning is presented as a nine-principle framework:
- **Metalearning** — learn how the subject is structured before committing to a path
- **Focus** — cultivate concentration as a skill
- **Directness** — practice the real activity, not only proxies for it
- **Drill** — isolate and attack bottlenecks
- **Retrieval** — use recall as a learning method
- **Feedback** — use external correction without ego defensiveness
- **Retention** — build spacing, overlearning, and memory supports
- **Intuition** — move beyond memorization into real understanding
- **Experimentation** — explore beyond standard methods once competence is established

The framework matters here because it generalizes ideas already emerging elsewhere in the wiki. It broadens [[wiki/concepts/problem-driven-learning]] beyond robotics and pairs it with a more explicit tactical toolkit: drills, retrieval, and feedback. It also intersects with [[wiki/concepts/deep-work]], since its intensity depends on the ability to sustain attention.

## Evidence & Examples

- The source defines ultralearning as "acquiring skills and knowledge in a self-directed and intense way" [[wiki/sources/ultralearning]]
- It recommends devoting roughly 10 percent of expected learning time to upfront metalearning research before starting [[wiki/sources/ultralearning]]
- It argues for project-based directness: code by writing programs, dance by dancing, and simulate real environments when direct immersion is impossible [[wiki/sources/ultralearning]]
- It treats retrieval, feedback, retention, and experimentation as distinct stages of improvement rather than afterthoughts [[wiki/sources/ultralearning]]

## Contradictions

> **Tension:** Ultralearning includes an explicit *metalearning* phase — map the subject first — while [[wiki/concepts/problem-driven-learning]] warns against overcommitting to roadmaps. These are not opposites so much as different failure-avoidance strategies: ultralearning says "map, then act"; problem-driven learning says "don't confuse the map with the path."
