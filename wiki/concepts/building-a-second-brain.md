# Building a Second Brain (BASB)

## Definition

A personal knowledge and project management system by Tiago Forte. A hybrid of (1) information management and (2) project management, built on the GTD tradition. Its goal is to offload cognitive overhead — "mind like water" — by externalizing information streams and project resources into a trusted system.

## The PARA System

The storage structure — four containers ordered by **action relevance** (urgency × importance):

| Container | Description | Time horizon |
|---|---|---|
| **Projects** | Active efforts with a goal and deadline | Short-term |
| **Areas** | Ongoing responsibilities to maintain a standard | Indefinite |
| **Resources** | Topics/interests that may become useful | Long-term |
| **Archive** | Inactive items from the above three | Indefinite |

Nothing has a permanent home; items move based on project relevance. This is deliberate — PARA is a "production system," not a filing system.

**Iceberg metaphor**: Projects float above the surface; Areas and Resources sit below, feeding projects as they mature into urgency. The system continually converts "important but not urgent" (Areas) into "important and urgent" (Projects).

## The CODE Workflow

The four-step process for handling incoming material:

1. **Capture** — fill inboxes with anything worth saving.
2. **Organize** — file into PARA by action relevance (project first, then area, then resource, then archive).
3. **Distill** — prepare resources via progressive summarization for later use.
4. **Express** — use the system to produce output (publish, implement).

## What BASB Is and Isn't

**BASB is**: resource management, project management, a system for managing information overload, a feeder for knowledge work.

**BASB is not**: knowledge processing. At no point in CODE do you move beyond the resource inside the system. Progressive summarization is resource preparation, not idea extraction. The actual knowledge work happens outside the system (in the manuscript, the Zettelkasten, the output).

> "The processing of knowledge, knowledge work, is explicitly not part of BASB." [[wiki/sources/building-a-second-brain-and-zettelkasten]]

## Who It's For

- People overwhelmed by information overload (openness-to-experience types drowning in inputs).
- People who lose sight of project-oriented work because they're too interested in the subject matter.
- Anyone who needs a trusted system for managing resources, tasks, and projects.

## Combination with Zettelkasten

BASB and ZKM address different parts of the knowledge value chain and combine naturally:
- **BASB/PARA**: manages resources, projects, tasks — the organizational layer.
- **ZKM**: processes knowledge, builds the integrated thinking environment — the intellectual layer.

See [[wiki/concepts/zettelkasten]] for the integration details.
